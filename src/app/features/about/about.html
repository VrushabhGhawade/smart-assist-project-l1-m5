Smart Assist – User Manual

Level 1 – Module 3 : Angular Essentials

1. About This Module

Module 3 focuses on mastering Angular essentials such as directives, pipes, component interaction, and UI frameworks. The Smart Assist application is enhanced with dynamic behavior, responsive layouts, and theming.

2. Learning Objectives

  • Understand and use Angular structural directives
  • Apply attribute directives for dynamic styling
  • Create and use custom directives
  • Work with Angular built-in and custom pipes
  • Build responsive UI using Angular Material and Bootstrap
  • Implement dynamic theme switching

3. Structural Directives

Structural directives are used to change the DOM structure dynamically.

  • if, else
  • switch, case, default
  • for, empty

4. Attribute Directives

Attribute directives change the appearance or behavior of elements.

  • [ngClass] – Apply CSS classes dynamically
  • [ngStyle] – Apply inline styles dynamically

5. Custom Directives

Custom directives are created to encapsulate reusable DOM behavior.

  • @HostBinding – Bind properties to host element
  • @HostListener – Listen to host events
  • ElementRef – Direct DOM element access

6. Angular Pipes

Pipes transform data for display in templates.

Built-in Pipes

  • currency
  • date
  • json
  • number
  • percent
  • keyvalue
  • slice
  • async

Pipe Types

  • Pure Pipes
  • Impure Pipes
  • Custom Pipes

7. Angular Material & Bootstrap

Smart Assist uses Angular Material and Bootstrap together for a clean, responsive UI.

  • Material Components: Button, Card, Toolbar, Form Controls
  • Bootstrap Grid System for layout
  • Mobile-first responsive design

8. Application Layout

  • Header – Application title & theme toggle
  • Left Panel – Navigation menu
  • Body – Dynamic content area
  • Footer – Developer info & user manual link

9. Dynamic Theme

The application supports dynamic theme switching to improve user experience.

  • Light & Dark theme support
  • Theme preference handled via service
  • Theme applied dynamically using classes

10. Local JSON Mock Data

All component data is connected to local JSON mock files to simulate backend interaction and support seamless data binding.

11. Evaluation Criteria

AC#1 – Functional & Responsive UI

  • Project blueprint is created
  • All components are created with hardcoded data
  • Application runs without errors
  • UI is responsive on desktop, tablet, and mobile

AC#2 – Code & Repository

  • Public GitHub repository is shared
  • Proper Angular project structure
  • Minimum 2–3 meaningful commits

12. How to Run the Application

          npm install
          ng serve

Open browser: http://localhost:4200
