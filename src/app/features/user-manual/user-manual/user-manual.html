Step 1: Login

  • Open the application to view the Login page.
  • Enter your User ID and Password.
  • Click Login.
  • The Login button is enabled only after entering valid details.

Step 2: Dashboard

  • You are redirected to a dashboard based on your role.
  • A welcome message with your name is displayed.
  • Use header menu for navigation.

Step 3: Roles & Access

  • User: Create tickets, view own tickets, track status.
  • Support Engineer: View assigned tickets.
  • Supervisor: Access reports and all tickets.
  • Unauthorized users cannot open restricted pages.

Step 4: Create Ticket

  • Click Create Ticket from the menu.
  • Fill all mandatory fields.
  • Submit button stays disabled until the form is valid.
  • After submission, ticket is created and you are redirected to Track Ticket page.
Mandatory Fields:
  • Issue Title
  • Description
  • Priority
  • Category
  • Sub-Category

Step 5: My Tickets

  • Click My Tickets to see all tickets you created.
  • Displayed using table view.
  • Click any ticket to open Track Ticket page.

Step 6: Track Ticket

  • View full ticket details.
  • Tickets can be selected from dropdown.
  • Direct access supported using URL parameter.

Example route: /track-ticket/123

Step 7: Live Chat

  • Displays informational placeholder page.
  • Future feature for chat support.

Step 8: Reports (Supervisor Only)

  • Only Supervisors can access this page.
  • Shows analytics and reports (future implementation).

Step 9: Navigation & Routing Features

  • Basic Routing
  • Nested Routing
  • Path Parameters
  • Query Parameters
  • Route Guards

Step 10: Forms & Validation

  • Reactive Forms used for ticket creation
  • Template Driven Forms supported
  • Field validations applied
  • Invalid forms cannot be submitted

  • Click Useful Resources from the footer.
  • This page provides documentation content organized into tabs: Requirement, Tips, Test Data, Evaluation Parameter.

  • Open the Useful Resources page and switch to the Requirement tab.
  • This tab separates Technical Requirements and Functional Requirements.
