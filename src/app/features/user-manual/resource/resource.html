Project Folder Structure – L1M4

The application follows a feature-based and scalable folder structure, separating core components, layouts, and feature modules. 

src/
 └── app/
     ├── core/
     │   ├── components/
     │   │   ├── header/
     │   │   └── footer/
     │   ├── services/
     │   │   ├── ticket.service.ts/
     │   │   └── user.service.ts/
     │   └── models/
     │
     ├── features/
     │   ├── auth/
     │   │   └── login/
     │   ├── admin/
     │   ├── support-engineer/
     │   ├── user/
     │   │   └── my-ticket/
     │   └── user-manual/
     │
     ├── layouts/
     │   ├── auth-layout/
     │   └── main-layout/
     │
     ├── app.routes.ts
     ├── app.component.ts
     └── app.config.ts
  • Core: Shared components like Header & Footer, and services like Auth service.
  • Features: Business modules such as Login, Admin, Support Engineer, User and User Manual
  • Layouts: Separate layouts for authentication and main app
  • Routing: Centralized route configuration in app.routes.ts
